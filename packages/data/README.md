# @milestone/data

## Purpose

Data layer for this monorepo: PostgreSQL connection, Drizzle schema, migrations, and repository **implementations** that satisfy interfaces from `@milestone/domain`.

The default implementation is **Drizzle ORM + Postgres**. Alternative ORM packages can be added later if they export the same `createDataLayer()` shape.

## What belongs here

- Drizzle table definitions (`src/schema/`)
- SQL migrations (`migrations/`, generated by drizzle-kit)
- Repository classes (`src/repositories/`)
- `createDataLayer({ connectionString })` factory in `src/index.ts`

## What does not belong here

- HTTP routes or auth middleware → `apps/api-primary`
- Pure domain types without persistence → `@milestone/domain`
- Shared React/UI code → `@milestone/js-common`

## Wiring to the API

`apps/api-primary` should instantiate the data layer at startup and pass repository methods into `createAuth()`:

```ts
import { createDataLayer } from "@milestone/data";
import { UserId } from "@milestone/domain";

const data = createDataLayer({ connectionString: process.env.DATABASE_URL! });

const auth = createAuth({
  findUser: (f) => data.users.findById(UserId(f.id)),
  verifyApiToken: (t) => data.apiTokens.verify(t),
});
```

## Environment

Copy `.env.example` to `.env` and set `DATABASE_URL`.

## Commands

```bash
npm run build
npm run db:generate   # create migration from schema changes
npm run db:migrate    # apply migrations
npm run db:push       # push schema directly (dev only)
npm run db:studio     # Drizzle Studio UI
```
